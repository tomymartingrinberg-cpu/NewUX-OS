Lo que hace distinta a NewUX OS

Pensada para quien ya pasó la curva de aprendizaje y quiere control real sobre su sistema, sin sacrificar pulido visual.

Tres kernels, una distro

Casi cualquier sistema operativo te encierra en un único kernel definido por su mantenedor. NewUX OS te deja elegir desde GRUB entre Mainline RC, Stable y LTS según necesites bleeding-edge, balance o estabilidad — más un modo Recovery siempre listo si algo se rompe.

Un solo gestor para todo

Los gestores tradicionales te obligan a aprender una herramienta distinta por cada formato — una para repositorios oficiales, otra para Flatpak, otra para AppImage, y todavía hay que compilar a mano lo que falta. pnu unifica todo eso bajo una API limpia tipo apt, y además auto-actualiza el kernel al último RC publicado en kernel.org.

Instalador que no oculta nada

La mayoría de los instaladores actuales son cajas negras: clickeás "siguiente" y rezás. El de NewUX OS está escrito en Python + GTK3, son cientos de líneas legibles, mostrá exactamente qué hace cada paso (particionado, cifrado LUKS opcional, configuración GRUB) y podés modificarlo vos mismo si te incomoda algo.

Identidad visual desde el primer arranque

Las distros típicas te dejan un escritorio con look genérico, esperando que pases horas configurando temas, fuentes, paneles, atajos y dock. NewUX OS viene con una paleta, una tipografía y un layout definidos y consistentes — no para limitarte, sino para darte un punto de partida que no parezca un demo de los noventa.

Tu propio repositorio sin infraestructura

Publicar un paquete para que tus usuarios lo instalen normalmente exige montar un servidor, firmar GPG, mantener un mirror y armar metadata compleja. Con NewUX OS te alcanza un repo git público con manifests YAML — un pnu repo-add del lado del usuario y ya está. Sin servidor propio, sin claves, sin ceremonia.

Reproducible, sin binarios mágicos

Demasiados sistemas operativos se distribuyen como imágenes opacas: bajás un blob de varios GB y nadie sabe cómo se generó. NewUX OS publica cada uno de los scripts y configuraciones que la construyen, paso a paso desde Linux From Scratch — podés auditar, modificar y rebuildear toda la distro desde cero en tu propia máquina.

Compilar desde cero

Bajás los scripts de build y reproducís toda la distro en tu host. Ideal para entender qué hay adentro o customizar pieza por pieza.

Tiempo 6–12 h Disco ~40 GB
Ver instrucciones →
Probar la ISO en QEMU UEFI: 
qemu-system-x86_64 -enable-kvm -cpu host -m 4096 -smp 2 \
  -drive if=pflash,format=raw,readonly=on,file=/usr/share/OVMF/OVMF_CODE_4M.fd \
  -drive if=pflash,format=raw,file=OVMF_VARS_4M.fd \
  -cdrom newuxos_v1.iso \
  -vga virtio -display gtk -device qemu-xhci -device usb-tablet
Grabar a USB: 
sudo dd if=newuxos_v1.iso of=/dev/sdX bs=4M status=progress oflag=sync
