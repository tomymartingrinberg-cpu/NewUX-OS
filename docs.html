NewUX OS Docs

Todo lo que necesitás para instalar, configurar, mantener y contribuir a NewUX OS. Si encontrás algo confuso o incompleto, abrí un PR — esta wiki es código en el repo.

Última actualización: v1.0.0 · 2026-06 · Editar en GitHub

Quick Start

Si querés simplemente probar NewUX OS en una VM, es 3 pasos:

  1. Descargá la ISO desde GitLab Releases (newuxos_v1.iso, 2.58 GiB).
  2. Booteala en QEMU UEFI: 
    qemu-system-x86_64 -enable-kvm -cpu host -m 4096 -smp 4 \
  -drive if=pflash,format=raw,readonly=on,file=/usr/share/OVMF/OVMF_CODE_4M.fd \
  -drive if=pflash,format=raw,file=OVMF_VARS_4M.fd \
  -cdrom newuxos_v1.iso \
  -vga virtio -display gtk
  3. En el menú GRUB, elegí NewUX OS Live y entrás al escritorio en ~20 segundos.
Probar sin compromiso: el modo Live no toca tu disco. Todo corre en RAM con overlayfs encima del squashfs. Para instalar al disco real desde la ISO live, abrí el icono Instalar NewUX OS del escritorio.

Instalación

Hay tres formas de poner NewUX OS en tu máquina:

1. ISO Live → instalar al disco

Es el flow recomendado para hardware físico:

  1. Grabá la ISO a un USB de 8 GiB+ con dd: 
    sudo dd if=newuxos_v1.iso of=/dev/sdX bs=4M status=progress oflag=sync
  2. Booteá desde el USB (entrá al firmware UEFI con F2/F12/Del según tu motherboard).
  3. Elegí "NewUX OS Live" en GRUB.
  4. Cuando carga el escritorio, ejecutá el icono Instalar NewUX OS.
  5. El instalador GTK3 te lleva por: idioma, disco, particionado, usuario, hostname, resumen, instalación, reboot.

2. Imagen booteable directa (.img.zst)

Si ya tenés el disco preparado:

zstd -d newux-os-1.0.img.zst -o /tmp/newux.img
sudo dd if=/tmp/newux.img of=/dev/sdX bs=4M status=progress oflag=sync

Esto sobreescribe todo el disco destino con un NewUX OS funcional. Sin preguntar.

3. Compilar desde cero (LFS)

Para usuarios que quieren auditar/customizar cada paso. Ver Build from source abajo.

Primer arranque

Después de instalar y reiniciar, NewUX OS te recibe con el greeter de LightDM. Ingresá con el usuario que creaste durante la instalación.

Una vez adentro vas a ver:

  • Top bar de 28px con menú aplicaciones (NewUX), windowmenu, systray, clock, botón de power.
  • Dock inferior centrado con 8 launchers: appfinder, archivos (Thunar), terminal, firefox, libreoffice, gimp, krita, vlc.
  • Wallpaper con gradiente navy-teal generado proceduralmente.

Lo primero que conviene hacer:

sudo pnu update                          # refresca metadata de repos
sudo pnu init-flatpak                    # configura Flathub si no está
pnu doctor                               # diagnostica el sistema

pnu — Packaging for NewUX Users

pnu es el gestor de paquetes universal de NewUX OS. Un solo binario bash (~1200 LOC) que unifica cuatro backends bajo una API estilo apt/dnf.

Instalar paquetes

El comando pnu install auto-detecta el backend según lo que le pases:

sudo pnu install firefox                       # busca en Flathub y repos NewUX
sudo pnu install firefox vlc inkscape          # múltiples
sudo pnu install --flatpak firefox             # forzar backend Flatpak
sudo pnu install --appimage https://obsidian.md/Obsidian-1.5.0.AppImage
sudo pnu install --source https://archive.gnome.org/gimp-2.10.36.tar.xz
sudo pnu install ./local-package.AppImage      # path local
sudo pnu install https://github.com/foo/bar.git  # git → source-compile

Actualizar

sudo pnu update                                # refresca metadata
sudo pnu upgrade                               # actualiza TODO
sudo pnu upgrade firefox                       # solo uno
pnu list upgrades                              # qué se va a actualizar

Remover paquetes

sudo pnu remove firefox                        # desinstala (mantiene config user)
sudo pnu purge firefox                         # desinstala + borra configs/cache
sudo pnu autoremove                            # huérfanos de Flatpak runtimes

Buscar / info

pnu search libreoffice                         # busca en todos los backends
pnu info firefox                               # detalles: versión, source, deps, tamaño
pnu list installed                             # qué tenés instalado
pnu which /usr/bin/firefox                     # qué paquete provee X archivo
pnu depends firefox                            # dependencias runtime
pnu stats                                      # resumen de tu sistema

Backend Flatpak (Flathub)

Detecta flatpak automáticamente. pnu install firefox busca en Flathub primero (org.mozilla.firefox). Si falta gpg en el sistema, configura --no-gpg-verify automáticamente.

Para inicializar Flathub manualmente:

sudo pnu init-flatpak

Backend AppImage

Descarga AppImages a /opt/appimages/, los hace ejecutables, genera un wrapper en /usr/local/bin/ y extrae el .desktop interno para que aparezca en el menú.

sudo pnu install --appimage https://github.com/obsidianmd/obsidian-releases/releases/download/v1.5.0/Obsidian-1.5.0.AppImage
sudo pnu install ./Obsidian-1.5.0.AppImage     # path local

Backend source-compile

Auto-detecta el sistema de build:

Soporta .tar.xz, .tar.gz, .tar.bz2, y URLs git directas.

Repositorios

Los manifests YAML del repo oficial NewUX viven en pnu-packages (también en GitLab).

Cómo NewUX maneja kernels

NewUX OS shippea con tres ramas de kernel disponibles desde GRUB:

  • Mainline RC — la versión release-candidate más nueva (ej. 7.1-rc5). Bleeding edge para hardware nuevo.
  • Stable — la última release estable (ej. 7.0.10).
  • LTS — long-term support (ej. 6.18.33). Más conservador.

Más una entrada de Recovery que arranca en modo single-user con el kernel marcado en /etc/pnu/recovery-kernel.

Auto-upgrade al último RC

Una de las features estrella de pnu:

sudo pnu update-kernel                         # auto-upgrade mainline (default)
sudo pnu update-kernel auto mainline           # explícito
sudo pnu update-kernel auto lts                # upgrade rama LTS
sudo pnu update-kernel auto stable             # upgrade rama Stable

Flujo interno:

  1. Consulta kernel.org/finger_banner y como fallback hace git ls-remote a https://git.kernel.org/pub/scm/linux/kernel/git/torvalds/linux.git.
  2. Detecta la última versión disponible para la rama (incluyendo RCs como 7.2-rc6).
  3. Si hay versión nueva: descarga el tarball, copia el .config actual, fuerza flags VIRTIO/EXT4/SERIAL, make olddefconfig, compila con make -j$(nproc), instala módulos.
  4. Setea el nuevo kernel como default de GRUB.
  5. Marca el kernel anterior de la misma rama en /var/lib/pnu/pending_kernel_removal.

Cleanup automático post-reboot

El service pnu-cleanup-old-kernels.service corre al boot siguiente, verifica que los kernels marcados no sean el running ni el recovery, y los elimina (/boot/vmlinuz-* + /lib/modules/* + config + System.map). Después regenera GRUB.

sudo systemctl enable pnu-cleanup-old-kernels.service     # activar
sudo pnu update-kernel cleanup                            # forzar manual
sudo pnu update-kernel remove 7.0.10                      # borrar específico

Modo Recovery

El kernel marcado como recovery nunca se borra automáticamente. Para configurarlo:

sudo pnu set-recovery 6.18.33                  # marcar LTS como recovery
cat /etc/pnu/recovery-kernel                   # ver cuál es

Cambiar el kernel default en GRUB

sudo pnu update-kernel menu                    # menú interactivo
sudo pnu update-kernel mainline                # default = última mainline instalada
sudo pnu update-kernel lts
sudo pnu update-kernel stable
pnu update-kernel list                         # lista instalados

Filosofía visual

El ricing de NewUX OS usa solo componentes XFCE estándar (sin whiskermenu, docklike, plank externos) para una experiencia tipo elementary/macOS:

  • Top bar minimalista 28px dark transparente.
  • Dock inferior centrado (panel-2 con length=1).
  • Acento teal #16c3a5 en buttons / focus / selection.
  • Wallpaper procedural navy-teal con mesh isométrico.

Aplicar el ricing

git clone https://github.com/tomymartingrinberg-cpu/newux-os-ricing
cd newux-os-ricing
sudo bash scripts/apply-ricing.sh

# Recargar XFCE
xfce4-panel -r &
xfdesktop --reload &

Personalizar

Los archivos de ricing se ubican en:

~/.config/xfce4/xfconf/xfce-perchannel-xml/   # config XFCE
~/.config/gtk-3.0/gtk.css                     # CSS GTK custom
~/.config/xfce4/terminal/terminalrc           # terminal
/usr/share/backgrounds/newux/                 # wallpapers

Para cambiar el acento, editá ~/.config/gtk-3.0/gtk.css y reemplazá @define-color newux_accent #16c3a5; por tu color. Reiniciá las apps GTK para ver el cambio.

Instalador internals

El instalador (install-newux-gui.py) está escrito en Python 3 + GTK3, ~1100 LOC, con CSS embebido para el branding. Hace los siguientes pasos:

  1. Splash con logo y bienvenida.
  2. Detecta discos con lsblk + blkid.
  3. Ofrece particionado automático (EFI 512M + swap 2G + root resto), alongside o manual.
  4. Opcional LUKS (requiere cryptsetup en el sistema live).
  5. Pide nombre, username, password, hostname.
  6. Resumen y confirmación.
  7. rsync del rootfs live al destino, chroot, grub-install --removable.
  8. Listo, opción de reboot inmediato.

Crear tu propio repo pnu

  1. Forkeá pnu-packages o creá un repo nuevo con la misma estructura.
  2. Agregá manifests YAML en packages/: 
    name: mi-app
version: 1.2.3
backend: appimage     # flatpak | appimage | source
source: https://example.com/mi-app-1.2.3.AppImage
maintainer: tu-usuario
license: MIT
categories: [Utility, Network]
description: |
  Descripción multi-línea.
homepage: https://example.com
  3. Subí a GitHub Pages / Netlify / Cloudflare Pages.
  4. El usuario lo agrega con sudo pnu repo-add tunombre https://tu.url.

Build from source (LFS)

Para reproducir NewUX OS completa desde cero (~6-12 hs en hardware moderno):

git clone https://github.com/tomymartingrinberg-cpu/NewUX-OS-build-system
cd NewUX-OS-build-system
sudo ./setup.sh                              # prepara host (dependencias)
sudo ./run-professional.sh                   # corre TODO el pipeline

El pipeline genera primero el cross-toolchain, después LFS base, BLFS (Xorg, XFCE, audio), nuestros tres kernels custom y finalmente aplica el ricing y empaqueta.

Requisitos: 30+ GiB libres, 4+ GiB RAM, sudo, gcc 11+, make, git, wget.

No arranca después de instalar

Síntoma típico: kernel panic VFS: Unable to mount root fs on unknown-block(0,0).

Causa habitual: grub-mkconfig generó la entrada con root=UUID=... pero el initrd custom de NewUX espera root=PARTUUID=... (sin initramfs convencional).

Fix manual: arrancá desde la ISO live, montá tu instalación, editá /boot/grub/grub.cfg en el disco, reemplazá root=UUID= por root=PARTUUID= (usá blkid /dev/sdaX para obtener el PARTUUID), y borrá la línea initrd /boot/initrd.img.gz si está.

Problemas gráficos

Si bootéa pero el escritorio no carga (pantalla negra, cursor X visible), probá:

  1. Editá la entrada GRUB (presioná e) y agregá nomodeset a la línea linux.
  2. Si usás NVIDIA propietario y tenés glitch, instalá los drivers correctos con sudo pnu install --source nvidia-driver-560.

Sin red después de instalar

NetworkManager debería arrancar por default. Verificá:

sudo systemctl status NetworkManager
sudo systemctl enable --now NetworkManager
nmcli device                                   # ver interfaces
nmcli device wifi list                         # WiFi disponibles
sudo nmcli device wifi connect "SSID" password "passwd"

Sin audio

systemctl --user status pipewire wireplumber
systemctl --user enable --now pipewire wireplumber
wpctl status                                   # estado WirePlumber

FAQ

¿Por qué basado en LFS y no en Debian / Arch / etc?

Porque querés transparencia total sobre lo que corre en tu máquina. LFS te obliga a construir cada paquete a mano, así que el sistema final es exactamente lo que vos definiste. No hay capas heredadas de decisiones de upstream.

¿Es estable para uso diario?

Es un proyecto experimental. Cosas básicas (browser, terminal, editor, gestión de archivos) funcionan bien. Pero todavía faltan piezas (Wayland, secure boot, hardening). No la pongas en producción de algo crítico.

¿Puedo instalar Steam / Wine / drivers NVIDIA?

Sí, vía pnu install --flatpak com.valvesoftware.Steam o desde Flathub directamente. Wine también vía Flatpak. NVIDIA propietario hay que compilar from source porque el repo es legacy de LFS.

¿Hay versión live persistente?

No por ahora — la ISO actual usa overlayfs tmpfs (cambios se pierden al reboot). Para persistencia, instalá al disco con el instalador GUI.

¿Soporta secure boot?

No todavía — el kernel y módulos no están firmados. Está en el roadmap.

¿Cómo reporto un bug?

Issues en GitHub: github.com/tomymartingrinberg-cpu/NewUX-OS/issues. Incluí: versión de NewUX OS, kernel running (uname -r), output de pnu doctor, pasos para reproducir.

¿Hay foro de comunidad?

Sí — NewUX Discuss. También GitHub Discussions del repo principal.

Contribuir

Las contribuciones son bienvenidas. Áreas que necesitan ayuda ahora mismo:

  • Manifests pnu — agregar más paquetes al repo oficial (pnu-packages).
  • Wayland — el kernel soporta DRM_VIRTIO_GPU pero el desktop default es Xorg.
  • Hardening — apparmor / SELinux integration.
  • Docs — traducciones, ejemplos, screencasts.

Workflow:

  1. Fork del repo correspondiente.
  2. Branch con tu cambio (git checkout -b mi-feature).
  3. Commits con mensajes claros (qué + por qué).
  4. PR contra main.
