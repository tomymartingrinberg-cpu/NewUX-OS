Cómo funciona

Usamos GitHub Discussions como backend — vos no necesitás aprender nada nuevo si ya tenés cuenta de GitHub.

1

Login con GitHub

Si ya tenés cuenta GitHub (o GitLab linkeado), un solo click. Si no, registrarte es gratis y sin email marketing.

2

Elegí categoría

Cada thread debe ir en su categoría. Postear "soporte" en "anuncios" te lleva a que te muevan el thread.

3

Buscá primero

Antes de iniciar un thread nuevo, usá la búsqueda. Probablemente tu pregunta ya fue contestada.

4

Sé claro y reproducible

Si reportás un bug, incluí: versión de NewUX OS, output de pnu doctor, pasos exactos para reproducir.

Reglas de convivencia

✓ Sí

  • Preguntas claras con contexto.
  • Compartir tu ricing, scripts, manifests pnu.
  • Reportar bugs con stack trace + reproducción.
  • Proponer features y justificar el use case.
  • Ayudar a otros usuarios.

✗ No

  • Spam, autopromoción agresiva.
  • Lenguaje discriminatorio o ataques personales.
  • Piratería, cracks, paquetes con malware.
  • Cross-posting (el mismo thread en 3 categorías).
  • "NewUX OS vs X distro" sin argumentos técnicos.
